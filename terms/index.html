Privacy Policy

Last Updated: January 2026
Version: 1

Chat App is designed to let users communicate using end-to-end encryption (E2EE). To support ongoing development, infrastructure, and storage costs, the app may display advertisements.

Aim

Our goal is to provide a secure platform where users can talk and connect without giving away unnecessary personal data. We believe privacy should be simple and automatic. That is why we collect only the minimum information required to create and operate your account, and nothing more.

We aim to protect user conversations through encryption and design our service so that we do not see, store, or sell readable personal data, while keeping the service operational and sustainable.

1. Purpose

TalkWe is primarily created for personal use, testing, and learning. Some limited commercial activity (such as advertising) may occur to help cover hosting, infrastructure, and data storage costs.

The main purposes of Chat App are:

  • Provide secure 1-on-1 and group messaging
  • Offer media sharing (images), protected by end-to-end encryption
  • Allow customizable public profile images and group images
  • Enable communication while prioritising user privacy

2. Data We Collect

We collect only the minimum information required for the app to function:

  • Email address (or phone number, if enabled) for login and account recovery
  • Display name
  • Date of birth (DOB) for age verification and basic personalization
  • Automatically generated UUID (unique internal user identifier)
  • Profile image (unencrypted and publicly visible, if provided)
  • Status message (public, if provided)

We do NOT collect:

  • Contact lists
  • Precise location data
  • Plain or readable message content
  • Browsing history
  • IP addresses for tracking purposes
  • Hardware identifiers beyond Firebase default requirements

3. Public vs. Private Data

Public Data (Not Encrypted)

The following information is visible to other users:

  • User profile image
  • Group profile image
  • Display name / username
  • Status message (if set)
  • Group name and description

Important: Do not upload sensitive or private profile photos. Profile and group images are stored unencrypted to allow visibility in chats and group lists.

Private Data (End-to-End Encrypted)

The following data is protected using E2EE:

  • One-on-one messages
  • Group messages
  • Shared chat images
  • Room encryption keys
  • User private encryption keys
  • Future document files (when available)

Only chat participants can decrypt private content.

4. User Consent

Media and messaging features are enabled by default. By sending messages, images, or other media through Chat App, you consent to:

  • End-to-end encrypted storage of that content
  • Secure delivery to intended recipients
  • Synchronization across your devices

If you do not agree, please avoid using messaging and media-sharing features.

5. Encryption and Message Security

User Keys

  • Each user is assigned a public and private encryption key
  • Private keys are encrypted using the user‚Äôs password
  • TalkWe cannot decrypt or access private keys

Room Keys

  • Each chat room uses a unique encryption key
  • Only authorized participants can decrypt room content

Message & Media Encryption

  • Messages and media are encrypted before being stored
  • Firebase and Cloudinary cannot read encrypted content
  • TalkWe developers cannot access user messages or media

üîê Passkey Protection (Optional)

To enhance account recovery while maintaining privacy, TalkWe provides an optional Passkey feature.

  • Passkeys are not enabled by default
  • Users must voluntarily create a Passkey in the app
  • Creating a Passkey is recommended but not mandatory

How Passkeys Work:

  • The Passkey protects access to encrypted private keys
  • During password recovery or re-login, users may be prompted to enter their Passkey
  • If the correct Passkey is provided, encrypted chats may be restored

Important Notice:

  • If a Passkey is not created, encrypted chats will be permanently lost after password recovery
  • If a Passkey exists but cannot be provided, encrypted data cannot be restored
  • TalkWe does not store Passkeys in readable form and cannot recover them
  • This design ensures encrypted data remains accessible only to the user

Password Recovery and Data Loss

  • Passwords can be changed only when the current (old) password is known
  • If the old password is not known, the password must be reset using account recovery
  • To restore encrypted chats after a password reset, a Passkey is required
  • If no Passkey exists, or it cannot be provided, all previous encrypted chats will be permanently lost
  • If a valid Passkey is provided, encrypted chats can be restored
