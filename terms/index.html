5. Encryption and Message Security

TalkWe uses a multi-layered end-to-end encryption (E2EE) system designed so that only intended recipients can read messages. TalkWe does not have access to readable messages, media, encryption keys, or backups.

User Encryption Keys

  • Each user has a public key and a private key
  • Private keys are securely stored in the Android Keystore
  • Private keys never leave the device
  • TalkWe cannot access or export private keys

Temporary Key Backup (Device Switching)

  • When switching devices, users may temporarily back up their encryption keys
  • This backup:
          1. Is encrypted using the user‚Äôs Passkey
          2. Is stored on the server for up to 48 hours
          3. Cannot be read by TalkWe (Passkeys are never stored)
  • Once recovery is completed, the temporary backup is deleted instantly
  • If the Passkey is lost or not provided, recovery is not possible

Chat Room Encryption

  • Each chat room has a unique secret room key
  • The room key secures message-level encryption keys
  • The room key is encrypted using each participant‚Äôs public key
  • Only users with the matching private key can unlock it

Message Encryption

  • Every message uses a new, unique message key
  • Message keys are protected by the room key
  • Messages are digitally signed to verify authenticity
  • Text messages are encrypted before transmission
  • Media files are converted into encrypted binary data using the same process

Server Role

  • TalkWe servers act only as a secure relay
  • Messages are temporarily held only for delivery
  • Messages are deleted once delivered to all participants
  • Undelivered messages are force-deleted after 5 days if a user remains inactive

This prevents long-term storage and avoids delivering outdated messages.

Backup Encryption

  • Local backups contain already encrypted chats
  • Backups are additionally encrypted using:
          1. A strong AES encryption key
          2. The AES key is encrypted using the user‚Äôs private key
  • Backup files remain unreadable without proper keys

Even if accessed externally, backups cannot be read.

7. Local Backup & Restore

Encrypted Local Backups

  • Users can manually create encrypted backups from within the app
  • Backups are generated as encrypted ZIP files
  • Users choose where to store the file
  • Backups are not automatic

Media Backup (Optional)

  • Users may choose to include media in backups
  • Including media significantly increases backup size
  • If disabled, media will not be restored

Restoring on a New Device

  • Users must manually transfer the backup file
  • During setup, the app prompts the user to select the file
  • The file is used to restore eligible local data

8. Device Switching & App Uninstallation

Switching to a New Device

To move your account safely:

1. Create a fresh local chat backup (media optional; size may increase)
2. Start the move process and create a Passkey
3. Log in on the new device, enter the Passkey, and locate the backup file

If completed correctly, chats and data are restored as if nothing changed. Temporary recovery data is deleted automatically after successful recovery.

Uninstalling the App Without Backup

  • All local chats are permanently lost
  • Encryption keys stored on the device are removed
  • Chats cannot be recovered, even if a backup file exists

This is a direct result of TalkWe‚Äôs privacy-first E2EE design.
